## 作业1：阅读 02-joint-bert-training-only 代码，并回答以下问题： bert 文本分类和 实体识别有什么关系，分别使用什么loss？ 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？

### 1. bert 文本分类和 实体识别有什么关系，分别使用什么loss？

*    **关系**
     *   **共享底层特征**：两者都利用 BERT 输出的上下文相关特征表示。文本分类通常使用 `[CLS]` 标记的输出向量作为整句语义表示；实体识别（NER）则使用每个 Token 对应的输出向量序列。
     *   **互补性**：文本分类提供的全局语义信息有助于实体识别消歧；实体识别提供的局部细粒度信息也有助于提升分类效果。
     *   **硬共享机制**：参数共享，梯度更新时会同时受到两个任务 Loss 的反向传播影响。

*   **Loss 函数**：
    *   **文本分类 (Text Classification)**：
        *   通常使用 **交叉熵损失 (CrossEntropyLoss)**。
        *   输入：`[CLS]` 位置的向量经过全连接层后的 logits。
        *   标签：句子级别的类别 ID。
    *   **实体识别 (Named Entity Recognition)**：
        *   通常使用 **Token 级别的交叉熵损失 (Token-wise CrossEntropyLoss)**。
        *   输入：每个 Token 位置的向量经过全连接层后的 logits 序列。
        *   标签：每个 Token 对应的实体标签 ID（如 B-PER, I-PER, O 等）。


### 2. 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？
    
简单的相加，可能存在以下问题：

*   **量级差异**：
    *   `seq_loss` 是基于句子数量的（Batch Size 个样本）。
    *   `token_loss` 是基于 Token 数量的（Batch Size × 平均句子长度 个样本）。
    *   由于 Token 数量远大于句子数量，`token_loss` 的数值通常远大于 `seq_loss`。直接相加会导致优化过程主要由 NER 任务主导，文本分类任务可能被“忽略”，导致分类准确率下降。
*   **收敛速度不一致**：
    *   不同任务的难易程度不同，收敛速度也不同。如果一个任务很快收敛而另一个还在剧烈波动，固定的权重相加会导致已收敛的任务被未收敛任务的梯度破坏，或者整体训练难以稳定。
*   **梯度冲突**：
    *   两个任务对共享参数（BERT 编码器）的梯度更新方向可能不一致。简单的线性叠加无法解决梯度冲突问题，可能导致模型在两个任务上都表现平平（负迁移）。
