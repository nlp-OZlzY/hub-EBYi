# 1. BERT文本分类和实体识别的关系及Loss
                                                                                
## 关系：
  
  这是一个**联合训练（Joint Training）**的多任务学习模型，同时完成两个任务：    
  
  - 意图识别（Intent Classification）：文本分类任务，判断整句话的意图         
  - 槽位填充（Slot Filling）：实体识别任务，识别句子中的关键信息


  #### 共享BERT编码器
  bert_output = self.bert(input_ids, attention_mask, token_type_ids)
  pooler_output = bert_output[1]  # [CLS] token表示，用于句子分类
  token_output = bert_output[0]   # 所有token的表示，用于序列标注

  #### 两个独立的分类头
  seq_output = self.sequence_classification(pooler_output)    # 意图分类
  token_output = self.token_classification(token_output)      # 槽位填充
  ####两个任务都使用 CrossEntropyLoss，但应用方式不同：
  - 意图识别：句子级别的多分类
  - 槽位填充：token级别的序列标注（使用BIO标注）

##Loss计算：

####  意图识别loss - 多分类交叉熵
  seq_loss = self.criterion(seq_output, seq_label_ids)
#### 实体识别loss - 序列标注交叉熵（只计算有效token）
  token_loss = self.criterion(active_logits, active_labels)
#### 简单相加
  loss = seq_loss + token_loss

# 2.多任务训练 loss = seq_loss + token_loss 的问题

  
1. Loss尺度不平衡
    - 两个任务的loss数值范围可能差异很大
    - 某个任务的loss可能主导梯度更新，导致另一个任务训练不充分
2. 收敛速度不一致
    - 简单任务（如意图分类）可能很快收敛
    - 复杂任务（如实体识别）可能需要更多训练
    - 简单相加会导致模型过早停止优化某个任务
3. 任务重要性无法调节
    - 实际应用中可能某个任务更重要
    - 简单相加无法体现业务优先级

##解决方案：
### 方案1：加权Loss

  # 手动设置权重
  alpha = 0.3  # 意图识别权重
  beta = 0.7   # 槽位填充权重
  loss = alpha * seq_loss + beta * token_loss
### 方案2：动态权重（Uncertainty Weighting）

  # 基于不确定性的自适应权重
  # 参考论文：Multi-Task Learning Using Uncertainty to Weigh Losses
  class MultiTaskLoss(nn.Module):
      def __init__(self):
          super().__init__()
          self.log_var_seq = nn.Parameter(torch.zeros(1))
          self.log_var_token = nn.Parameter(torch.zeros(1))

      def forward(self, seq_loss, token_loss):
          precision_seq = torch.exp(-self.log_var_seq)
          precision_token = torch.exp(-self.log_var_token)
          loss = precision_seq * seq_loss + self.log_var_seq + \
                 precision_token * token_loss + self.log_var_token
          return loss
### 方案3：Loss归一化

  # 对loss进行归一化，使其在相同尺度
  seq_loss_norm = seq_loss / seq_loss.detach()
  token_loss_norm = token_loss / token_loss.detach()
  loss = seq_loss_norm + token_loss_norm
### 方案4：梯度归一化（GradNorm）

  # 动态调整权重，使各任务梯度范数保持平衡
  # 需要在训练过程中监控各任务的梯度并调整权重
### 方案5：分阶段训练

  # 先训练一个任务，再联合训练
  if epoch < 3:
      loss = seq_loss  # 先训练意图识别
  else:
      loss = seq_loss + token_loss  # 再联合训练
### 解决思路：
  - 先尝试加权Loss（方案1），根据验证集表现调整权重
  - 如果两个任务差异很大，使用动态权重（方案2）
  - 监控训练过程中两个任务的loss曲线，确保都在下降
