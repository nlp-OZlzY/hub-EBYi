
# 智能FAQ客服系统设计方案

## 目标
构建一个基于语义匹配的智能问答系统，替代人工客服处理常见问题。用户提问时，系统自动匹配历史FAQ中最相似的问题，并返回对应答案。

## 1. 数据库设计
需设计结构化数据库存储FAQ及FAQ类目数据

## 2. 核心技术选型

### 使用模型：BERT（Sentence-BERT）
- 使用模型：`bert-base-chinese` 
- 使用 Sentence-BERT 架构进行句子编码，提升语义相似度计算效果。
- 将所有 FAQ 问题预先编码为 768 维向量，存入**向量数据库**（如 Faiss、Annoy 或 Milvus）。

### 匹配流程：
1. 用户输入问题 → 文本清洗（去噪、标准化）
2. 使用 BERT 编码为向量
3. 在向量库中检索 Top-K 最相似问题（余弦相似度）
4. 若最高相似度 > 阈值（如 0.8），返回对应答案；否则转人工或提示“未找到”

## 3. 是否需要大模型？
- **当前阶段无需引入大模型**（如通义千问）生成答案，以保证准确性和响应速度。
- 可后续引入大模型用于：
  - 问题同义改写，增强召回
  - 自动构建FAQ对
  - 多轮对话理解（进阶功能）

## 4. 系统优势
- 轻量高效，适合私有化部署
- 准确率高，基于语义而非关键词
- 支持持续迭代更新FAQ库

## 5. 后续优化方向
- 加入用户反馈机制，持续优化匹配效果
- 定期重训/更新向量库
- 结合分类模型先做类目预测，缩小检索范围
