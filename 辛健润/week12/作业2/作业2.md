#  1. 什么是前后端分离？

 前后端分离是一种开发模式，它把一个网站或应用的工作分成两个独立的部分：
   -  前端：负责用户能看到和操作的部分（界面、按钮、动画、输入框等）。
   -  后端：负责在背后处理数据、逻辑和存储的部分（比如用户注册、发布文章、读取数据库等）。

 在项目中：
   -  前端是 Streamlit，负责页面、输入输出、会话展示。
   -  后端是 FastAPI，负责用户/聊天/股票等业务和大模型调用。
   -  二者通过 REST通信：普通查询用 HTTP，聊天回复用 text/event-stream 流式输出。


# 2. 历史对话如何存储，以及如何作为下一次模型输入？

  这个项目里有两层存储：

  - 业务层存储（用于列表、回看、反馈）：assert/sever.db
      - 表：chat_session（会话元信息）和 chat_message（每条消息）。
      - 每次请求先存用户消息、结束后存 assistant 消息。
      - 前端切换历史会话时，调用 /v1/chat/get 拉取历史并显示。
  - 模型记忆层存储（用于下一轮上下文）：assert/conversations.db
      - 后端为每次会话创建 AdvancedSQLiteSession。
      - 调模型时把这个 session 传入 Runner.run_streamed(...)，框架会按同一 session_id 自动读写历史上下文，所以下一轮会带上之前对话。
      - 前端每次请求都会带 session_id（首次通过 /v1/chat/init 获取），从而命中同一历史。

   sever.db 负责“业务可见历史”，conversations.db 负责“模型可用记忆”；下一次输入靠同一个 session_id 自动续上下文。
