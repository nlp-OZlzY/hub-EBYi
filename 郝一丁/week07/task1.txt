1. bert 文本分类和实体识别有什么关系，分别使用什么 loss？

二者都是基于同一个 BERT 编码器完成的下游任务，只是预测粒度不同。文本分类是句子级任务，用来判断整句话的类别或意图；实体识别是 token 级任务，用来判断句子中每个词或字的标签。它们共享底层语义表示，因此可以联合训练。
在这份代码里，两者使用的都是交叉熵损失。文本分类对应的是句子级的 seq_loss，实体识别对应的是 token 级的 token_loss。其中实体识别只对有效 token 计算损失，不把 padding 部分算进去。



2. 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？

直接相加的坏处是默认两个任务同等重要，但实际上两个任务的难度、收敛速度和 loss 大小可能并不一样。尤其实体识别是 token 级任务，通常监督信号更多，更容易主导训练，导致文本分类学得不够好。此外，两个任务共享同一个 BERT，也可能出现梯度互相干扰的问题。
如果出现训练不平衡，最常见的处理方法是给两个 loss 加权，不再直接相加，而是让重要性更高或更难训练的任务占更合适的比例。进一步还可以用动态权重、分阶段训练，或者使用专门的多任务平衡方法来缓解任务之间的冲突。
