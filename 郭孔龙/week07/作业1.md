### 作业1: 阅读 02-joint-bert-training-only 代码，并回答以下问题：

#### bert 文本分类和 实体识别有什么关系，分别使用什么loss？

* 文本分类是句子级别的任务，实体识别是token级别的任务，前者判断意图，后者抽取结构化信息，相辅相成，文本分类使用seq_loss，实体识别使用token_loss

#### 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？

* 简单相加的坏处：
  * 量级不平衡问题
    * 原因分析：
      * 文本分类：只预测 1 个标签，loss 较大
      * 实体识别：预测 N 个 token 标签并取平均，loss 较小
      * 两个任务的 loss 量级可能差异很大
      * 结果：意图识别很好，槽位填充很差
    * 梯度冲突问题
      * 两个任务的梯度方向可能相反
      * 结果：梯度相互抵消，训练变慢
    * 收敛速度不同
      * 任务 A 收敛快，任务 B 收敛慢
      * 问题：如果此时停止训练，槽位填充效果会很差
    * 样本数量不平衡
      * 虽然 token 数量多，但每个 token 的 loss 很小
      * 导致：token_loss 在总 loss 中占比过小
* 解决方案
  * 对于本项目：
    * 第一步：先使用固定权重（快速验证）
    * 第二步：监控训练过程
    * 第三步：根据效果调整
      * 如果意图识别好，槽位填充差 → 增加 token_weight
      * 如果槽位填充好，意图识别差 → 增加 seq_weight
    * 第四步：如果想追求最佳效果 → 使用不确定性加权
      * 修改 model.py 添加可学习的不确定性参数
      * 修改 main.py 使用自动加权
      * 不需要手动调参，模型自动平衡
      * 这样可以让模型自己学会"哪个任务更难，就多关注哪个任务"！
