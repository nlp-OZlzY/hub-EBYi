### 问题一：

> 前后端分离是指将系统划分为前端和后端两个相对独立的部分。前端负责页面展示、用户交互和请求发送，后端负责业务逻辑处理、数据库访问、模型调用以及接口返回。二者通过 HTTP API 进行通信。在该项目中，后端基于 FastAPI 实现，通过 `/v1/chat`、`/v1/user`、`/v1/data`、`/v1/stock` 等接口向前端提供服务。

### 问题二：

> 该项目中的历史对话采用了双重存储机制。
>  第一，项目将聊天会话和消息内容存储在关系型数据库中，主要包括 `ChatSessionTable` 和 `ChatMessageTable`。其中，`ChatSessionTable` 记录会话基本信息，如 `user_id`、`session_id` 和标题；`ChatMessageTable` 记录每条消息的角色、内容、创建时间和反馈信息。这样可以支持历史对话的查询、删除、列表展示和反馈管理。
>  第二，项目还使用 `AdvancedSQLiteSession` 将会话状态存储到 SQLite 文件 `./assert/conversations.db` 中，用于维护 Agent 的上下文记忆。
> 当用户再次发起对话时，前端会继续传入相同的 `session_id`。后端在调用大模型时，通过 `Runner.run_streamed(agent, input=content, session=session)` 将当前输入 `content` 与对应 `session_id` 的会话记忆一起传入模型。这样，大模型就能够结合历史对话内容进行连续多轮交互。
