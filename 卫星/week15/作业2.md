| 维度              | MinerU                               | pdfplumber                                 |
| :---------------- | :----------------------------------- | :----------------------------------------- |
| 原生文本 PDF      | 能抽取，并会重组为 Markdown/JSON     | 很适合，速度轻，结果可控                   |
| 扫描件/图片型 PDF | 有 OCR 能力                          | 基本不行，需要另接 OCR                     |
| 多栏论文          | 更擅长恢复阅读顺序                   | 容易按坐标顺序串乱，需要调参               |
| 公式              | 可转 LaTeX                           | 通常只能抽到零散字符，数学结构差           |
| 表格              | 可转 HTML，偏结构化文档输出          | 对规则表格不错，但复杂/跨页/无线表格需调参 |
| 页眉页脚/页码     | 有自动清理思路                       | 通常要自己按区域过滤                       |
| 输出形态          | Markdown、JSON、中间结构、可视化结果 | 文本、表格列表、字符/线框对象              |
| 部署成本          | 重，模型/服务/API 依赖更多           | 轻，一个 Python 包即可                     |
| 可控性            | 高层结果更方便，但模型行为更黑盒     | 底层对象透明，适合精细调试                 |

- MinerU 的效果优势主要体现在“复杂文档”：论文、双栏、多图表、公式、扫描件、跨页表格、需要直接进 RAG 的文档。
- pdfplumber 的优势是“简单、透明、工程可控”：如果 PDF 本身是电子文本，表格线清楚，且只需要抽正文或固定区域数据，pdfplumber 往往更快、更稳定，也更容易写规则修正。
