# AI 狼人杀 — 多智能体协作与博弈的 Agent Team 系统

## 项目简介

AI 狼人杀是一个基于多 Agent 协作框架构建的智能博弈系统，项目以狼人杀为核心场景，探索多智能体在信息不对称环境中的协作、对抗、推理与决策能力。

系统通过构建多个拥有不同身份、目标和信息权限的 AI Agent，使其在严格信息隔离的条件下，自主完成夜晚行动、白天发言、投票放逐、身份推理和阵营博弈等行为。项目不仅关注“AI 能否玩狼人杀”，更关注复杂 Agent Team 系统中的状态管理、协作机制、策略生成、推理链路和全过程可观测性。

整个系统采用前后端分离架构，后端负责对局引擎、多 Agent 调度、日志追踪与评测复盘，前端负责实时观战、Agent 行为展示、时间线回放与结果可视化。系统核心推理能力基于 Qwen API 构建，通过统一的大模型调用层驱动所有角色 Agent 的思考与决策。

本项目特别强调：

* 多 Agent 协作与对抗
* Agent 可观测性
* 单元测试驱动开发
* 集成测试闭环验证
* Agent 行为评测
* 结构化日志追踪
* 自进化策略优化

本项目适合作为：

* 多 Agent 系统实战项目
* Agent Team 协作框架实践
* LLM 博弈推理研究
* AI 游戏智能体项目
* Agent 可观测系统
* Prompt 工程实验平台
* 大模型评测平台
* 自进化 Agent 实验项目

---

# 项目目标

本项目旨在构建一个完整的多 Agent 博弈系统，实现从角色生成、状态流转、Agent 推理、阵营协作到评测复盘的完整闭环。

系统重点解决以下问题：

* 多 Agent 如何在严格信息隔离下协作与对抗；
* 如何构建支持复杂规则的狼人杀对局引擎；
* 如何实现不同角色 Agent 的差异化策略；
* 如何实现狼人阵营的协同欺骗；
* 如何实现好人阵营的逻辑推理；
* 如何记录 Agent 的完整思考与行动链路；
* 如何实现 AI 对局的全过程可观测；
* 如何基于日志实现评测与复盘；
* 如何构建 Agent 的测试体系；
* 如何让 Agent 在多轮对局中持续优化。

# 核心功能

## 多角色 Agent 系统

系统内置多个狼人杀角色 Agent，每个 Agent 都拥有独立的身份、目标、策略和行动空间。

支持角色包括：

* 狼人 Agent
* 预言家 Agent
* 女巫 Agent
* 猎人 Agent
* 守卫 Agent
* 平民 Agent
* 法官 Agent

不同角色拥有不同的信息可见范围与行动权限。例如狼人可以知道同伴身份，预言家拥有查验能力，女巫拥有解药与毒药，而平民只能通过公开信息进行推理。

每个 Agent 都会基于当前局势、自身身份、历史发言和阵营目标生成策略与行为。

---

## 信息隔离机制

系统采用严格的信息隔离机制，确保每个 Agent 只能访问其权限范围内的信息。

系统中的信息分为：

* 公共信息
* 私有信息
* 阵营信息
* 行动结果信息
* 历史记忆信息

所有 Agent 无法直接读取全局真实状态，只能根据自身可见信息进行推理与决策，从而保证狼人杀博弈的真实性。

---

## 狼人杀对局引擎

系统构建完整的狼人杀游戏引擎，用于驱动整个对局流程。

引擎负责：

* 角色分配
* 回合推进
* 夜晚行动
* 白天讨论
* 投票放逐
* 技能执行
* 死亡结算
* 胜负判定
* 状态同步

整个系统围绕：

* 夜晚阶段
* 白天阶段
* 投票阶段

进行循环推进，并在满足阵营胜利条件时自动结束对局。

---

## Agent 推理与决策

每个角色 Agent 都会基于当前局势生成自己的策略与行动。

Agent 能够完成：

* 身份分析
* 发言生成
* 投票决策
* 可疑对象判断
* 阵营可信度评估
* 欺骗与伪装
* 战术协作
* 关键角色保护

狼人阵营会尝试误导与隐藏身份，好人阵营则会尝试基于逻辑与信息进行身份推断。

系统支持记录完整的 Agent 推理过程，用于后续评测与复盘。

---

## Qwen API 多模型推理

项目基于 Qwen API 构建所有 Agent 的核心推理能力。

不同角色 Agent 会调用统一的大模型推理层生成：

* 发言内容
* 投票策略
* 身份推理
* 阵营判断
* 夜晚行动
* 战术分析

系统支持：

* Qwen Plus
* Qwen Max
* Qwen Turbo

等不同模型版本，用于不同复杂度和成本场景。

统一的大模型调用层负责：

* Prompt 管理
* 模型路由
* JSON 输出解析
* 错误重试
* Token 控制
* 推理日志记录

从而避免各个 Agent 直接耦合具体模型接口。

# 技术架构

## 前后端分离架构

项目采用前后端分离架构设计。

后端负责：

* 狼人杀规则引擎
* 多 Agent 调度
* Qwen API 调用
* 推理与决策
* 记忆管理
* 信息隔离
* 日志追踪
* 评测复盘

前端负责：

* 实时观战
* Agent 发言展示
* 游戏状态展示
* 时间线回放
* Leaderboard 可视化
* 评测结果展示

---

## 通信架构

系统采用：

* REST API
* SSE（Server-Sent Events）

作为系统通信方案。

REST API 用于普通业务请求。

SSE 用于：

* 对局实时事件推送
* 发言流展示
* 状态变化广播
* 时间线同步

相比 WebSocket，SSE 更适合狼人杀这类以“服务端事件广播”为核心的多 Agent 场景，系统更加轻量、稳定且易于调试。

# 测试驱动开发（TDD）

本项目强调：

* 每完成一个 Agent 即进行单元测试
* 每完成一个核心模块即进行模块测试

整个项目采用测试驱动开发（TDD）思想构建。

---

## Agent 单元测试

每个角色 Agent 在开发完成后，都需要进行独立单元测试。

测试目标包括：

### 身份行为正确性

验证：

* 狼人是否能够正确协同
* 预言家是否能够正确查验
* 女巫是否能够正确使用药剂
* 猎人是否能够正确开枪
* 守卫是否能够正确守护

---

### 推理逻辑正确性

验证：

* Agent 是否能够正确分析发言
* 是否能够根据历史信息更新怀疑对象
* 是否能够进行合理投票

---

### 信息隔离正确性

验证：

* Agent 是否只能读取权限内信息
* 是否存在全局信息泄漏
* 是否存在非法状态访问

---

### 输出结构正确性

验证：

* JSON 输出是否合法
* 推理字段是否完整
* 行动格式是否符合协议
