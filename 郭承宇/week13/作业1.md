# 作业1

## V3 相比 V2 的改进

1. 架构创新
   - V3 移除了 V2 中的辅助损失函数，开创了 "无辅助损失负载均衡(Auxiliary-loss-free)" 它为每个专家设置一个动态调整的偏置项，若某专家过载或欠载，就动态增减这个偏置，进而引导路由的选择，这样既保持了负载均衡，又避免了对模型能力的损害
   - V3 增加了 "多token预测(Multi-token Prediction,MTP)"，让模型在每个位置预测多个token，增强了训练信号的密度，促使模型学习更长程的规划能力，从而提升模型性能

 2. DeepSeekMoE改进
    - 在 V2 共享专家和细粒度路由专家的基础上，V3 将亲和度计算从 Softmax 改为 Sigmoid，并对门控机制进行了归一化处理，使得专家选择更具区分度，有助于更精细的任务分配
   
3. 训练效率提升
   - V3 首次在超大规模模型上全面验证了 FP8 精度训练，配合精细量化和高精度累加，在保证精度的同时，大幅度提升了计算速度并降低显存占用
   - V3 设计了 DualPipe 算法，通过精巧调度实现计算和通信近乎完美的重叠，攻克了跨节点专家并行的沉重通信负担
   - 得益于上面的优化，V3 在 14.8T 数据上的预训练成本仅约为 266.4 万 H800 GPU 小时，全量训练也仅 278.8 万小时

  4. 后训练与推理能力提升
     - V3 通过专家模型生成、拒绝采样等多阶段流程，将 R1 系列模型的 CoT 推理能力蒸馏到自身，使得 V3 在推理任务上实现了大幅度增强

  5. 部署创新
     - 针对 MoE 模型在线推理时热点专家导致对应 GPU 过载的问题，V3 提出冗余专家策略，系统每 10 分钟统计一次各专家负载，动态复制高负载专家到空闲 GPU，这一工程方案有效消除了热点瓶颈，保障了高并发下的服务稳定性。
