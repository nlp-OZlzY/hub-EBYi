# 作业2：MinerU vs PaddleOCR pp_structurev3 文档解析效果对比

## 1. 测试环境

- **测试文档**：同一张包含中英文混排的新闻图片（厄立特里亚孔子学院报道）
- **PaddleOCR版本**：pp_structurev3
- **MinerU版本**：Docker部署版本

---

## 2. 输出格式对比

| 特性 | PaddleOCR pp_structurev3 | MinerU |
|------|-------------------------|--------|
| **Markdown输出** | ✅ 支持 | ✅ 支持 |
| **JSON结构化输出** | ✅ 支持（含bbox坐标） | ✅ 支持（含bbox坐标+模型配置） |
| **DOCX输出** | ❌ 不支持 | ✅ 支持 |
| **LaTeX输出** | ❌ 不支持 | ✅ 支持 |
| **可视化图片** | ❌ 不提供 | ✅ 提供多种（布局检测、OCR、预处理等） |
| **图片提取** | ✅ 保存到images目录 | ✅ 保存到imgs目录 |

**结论**：MinerU输出格式更丰富，支持DOCX和LaTeX导出，并提供多种可视化调试图片。

---

## 3. 文本识别质量对比

### 3.1 标题识别

**原文**：助力双方交往 搭建友谊桥梁

| 工具 | 识别结果 | 评价 |
|------|---------|------|
| PaddleOCR | `助力双方交往 搭建友谊桥梁` | ✅ 完全正确，保留空格 |
| MinerU | `助力双方交往搭建友谊桥梁` | ⚠️ 缺少空格 |

### 3.2 正文段落识别

**原文片段**：身着中国传统民族服装的厄立特里亚青年依次登台表演中国民族舞、现代舞、扇子舞等，曼妙的舞姿赢得现场观众阵阵掌声。

| 工具 | 识别结果 | 评价 |
|------|---------|------|
| PaddleOCR | 完整保留原文，无缺失 | ✅ 优秀 |
| MinerU | "身着中国传统民族服装的厄立特里亚**青次登台**表演中国民族舞、现代舞、扇子舞**曼妙的舞姿**..." | ❌ 多处文字缺失 |

**MinerU文字缺失示例**：
- "青年**依次**登台" → "青**次**登台"（缺失"年"和"依"）
- "扇子舞**等**，曼妙" → "扇子舞曼妙"（缺失"等"和逗号）

### 3.3 引号处理

| 工具 | 处理方式 | 评价 |
|------|---------|------|
| PaddleOCR | 使用中文引号"" | ✅ 符合中文排版规范 |
| MinerU | 混合使用中英文引号 | ⚠️ 不够统一 |

## 4. 结构化信息对比

### 4.1 PaddleOCR JSON结构

```json
{
    "type": "text",
    "text": "助力双方交往 搭建友谊桥梁 ",
    "text_level": 1,  // 标题层级
    "bbox": [85, 26, 908, 93],  // 边界框坐标
    "page_idx": 0
}
```

**特点**：
- 提供`text_level`字段标识标题层级
- 坐标格式简洁
- 结构简单直接

### 4.2 MinerU JSON结构

```json
{
    "block_label": "doc_title",
    "block_content": "助力双方交往搭建友谊桥梁",
    "block_bbox": [76, 0, 1400, 97],
    "block_id": 0,
    "block_order": 1
}
```

**特点**：
- 使用`block_label`分类（doc_title、text、paragraph_title、image、figure_title等）
- 提供`block_order`标识阅读顺序
- 包含完整的模型配置信息

## 5. 图片处理对比

| 特性 | PaddleOCR | MinerU |
|------|-----------|--------|
| **图片提取** | ✅ 提取到images目录 | ✅ 提取到imgs目录 |
| **图片引用方式** | Markdown语法 `![](images/xxx.jpg)` | HTML语法 `<img src="imgs/xxx.jpg">` |
| **图片描述** | ✅ 提供AI生成的图片描述 | ❌ 不提供 |
| **图片位置** | 文末集中展示 | 文中按位置插入 |

**PaddleOCR图片描述示例**：
```html
<details>
<summary>natural_image</summary>
Two girls smiling while drawing at a table outdoors, one wearing a headband and the other in casual clothing (no visible text or symbols)
</details>
```

## 6. 版式还原能力

### 6.1 PaddleOCR
- 按照原文阅读顺序排列段落
- 保留段落间的空行分隔
- 标题使用Markdown语法（#）

### 6.2 MinerU
- 图片位置更准确（插入在对应文字旁边）
- 提供图片标题（figure_title）的独立识别
- 但存在段落合并问题（多段文字可能被合并）

## 8. 结论与建议

### 8.1 选择建议

| 使用场景 | 推荐工具 | 原因 |
|---------|---------|------|
| **文本提取为主** | PaddleOCR | 文字识别准确率更高 |
| **需要DOCX/LaTeX输出** | MinerU | 输出格式更丰富 |
| **调试和可视化** | MinerU | 提供多种检测结果图 |
| **批量处理** | 两者皆可 | 都支持命令行批量处理 |
| **高精度要求** | PaddleOCR | 文字缺失更少 |

### 8.2 总结

1. **PaddleOCR pp_structurev3** 在**文本识别准确性**上表现更好，特别是对于中文文本的完整保留，几乎无文字缺失。

2. **MinerU** 在**输出格式多样性**和**调试便利性**上更胜一筹，提供DOCX、LaTeX导出以及多种可视化图片，适合需要深度分析和调试的场景。

3. 两者各有优势，建议根据具体需求选择：
   - 追求文本质量 → PaddleOCR
   - 追求功能丰富 → MinerU
