## 1.1 bert 文本分类和实体识别有什么关系，分别使用什么loss？
> + 文本分类：对整个句子或文档预测一个类别标签
>    实体识别：对输入序列的每个 token 预测一个标签
> + 文本分类：输出是 [batch_size,num_labels] ,表示句子属于各个类别的概率
>    实体识别：输出是 [batch_size,seq_len,num_labels] ,表示每个token属于各个标签的概率
> + 文本分类：使用的是 token 的最终隐藏状态(pooler_output)
>    实体识别：使用的是每个 token 的隐藏状态
> 
> ---
> 
> 文本分类的损失函数：Cross Entropy Loss 交叉熵损失
> 实体识别的损失函数：Cross Entropy Loss 交叉熵损失，但是需要忽略填充位置的损失
>


## 1.2 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，应该如何处理？
> + 坏处：
>   + 意图分类损失通常是句子级的交叉熵，而槽填充损失通常是 token 级交叉熵，token 级交叉熵可鞥因为序列长度累积而远大于句子级交叉熵，导致模型训练时更关注槽填充任务，而忽略意图分类
>   + 不同任务的收敛速度不同，简单相加会导致梯度更新被快速收敛的任务主导
> 
> ---
> 
> 处理方法：
> 1. 加权求和：通过引入权重(a,b)来平衡二者的损失  loss = a * seq_loss + b * token_loss
> 2. 对每一个任务的梯度分别进行归一化之后再相加
>
