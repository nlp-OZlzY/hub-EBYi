# 多模态RAG图文问答系统 - 架构设计

## 项目概述

搭建多模态RAG图文问答系统，依托图文混合PDF知识库，实现图文联动智能问答，解决纯文本问答无法结合图片、图表内容推理作答的痛点。

## 模块说明

### 1. 文档解析流程 (Worker)

MinerU解析PDF后：
- **文本**: 切分chunk → BGE embedding → 存入Milvus
- **图片**: CLIP embedding(整张) → 存入Milvus

### 2. 问答流程 (RAG)

1. 用户提问 → **BGE** 文本检索 → 召回相关文本chunks
2. 用户提问 → **CLIP** 图像检索 → 召回相关图片
3. 文本+图像融合 → **Qwen-VL** 推理
4. 返回带来源标注的答案

## 接口定义

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/upload` | POST | 上传PDF，返回知识库ID |
| `/api/chat` | POST | 多模态问答 |
| `/api/knowledge/{id}` | GET | 获取知识库状态 |
| `/api/worker/status` | GET | Worker状态 |

## 技术选型

| 组件 | 技术 |
|------|------|
| PDF解析 | MinerU |
| 文本嵌入 | BGE (bge-m3) |
| 图像嵌入 | CLIP (clip-vit-large-patch14) |
| 向量存储 | Milvus |
| 多模态LLM | Qwen-VL |
| 消息队列 | Kafka |
| 关系数据库 | MySQL |
| 后端框架 | FastAPI |
| 前端 | Vue3 + Vite + TypeScript |
