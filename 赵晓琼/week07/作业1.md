## Bert文本分类和实体识别有什么关系？分别用什么loss？多任务训练loss=seq_loss + token_loss有什么坏处？如果存在训练不平衡的情况，如何处理？

### Bert文本分类和实体识别的关系及loss
- Bert分类和实体识别都属于基于BERT的自然语言处理(NLP)下游任务，核心是共享BERT的预训练能力，都是用BERT提取文本的深层语义特征，但任务目标和输出形式完全不同。二者即存在底层技术的共通性，也有任务逻辑的本质差异。

  - 文本分类(预测整个文本的类别BertForSequenceClassification)：

    - 输入：single sentence:[CLS], w1, w2, w3, ... -> BERT -> [CLS]_vec -> Linear Classifier -> class

    - 输出：class(一个标签,如正面/负面，体育/科技等)

    - 使用的loss:单标签分类(每个样本只有一个类别)用CrossEntropyLoss;多标签分类(每个样本可能有多个类别)用BCEWithLogitsLoss(带sigmoid的二分类交叉熵损失)

  - 实体识别(需要为输入序列的每一个token都进行预测BertForTokenClassification)：

    - 输入：single sentence: [CLS], w1, w2, w3 ... -> BERT -> [w1_vec, w2_vec, w3_vec, ...] -> [Linear Cls, Linear Cls, Linear Cls, ...] -> [class1, class2, class3, ...]

    - 注意：虽然上部写了三个Linear Cls,实际实现中通常是一个共享权重的线性层，作用于每个token向量上。

    - 输出：class of each word(一个输入序列等长的标签序列)

    - 使用的loss：CrossEntropyLoss交叉熵

### 多训练任务loss=seq_loss + token_loss 有什么坏处？如果存在不平衡的情况，如何处理？

- 多训练任务loss=seq_loss + token_loss 有什么坏处？
  
  - 梯度主导问题：假如token_loss数值远大于seq_loss，假设token_loss=5，seq_loss=0.5，总loss主要由大数值的那个任务主导。反向椽笔时，剃度主要为了优化那个大Loss的任务，导致小Loss的任务几乎不学习，模型可能变成“只做实体识别”或“只做意图识别”的偏科生。

  - 收敛速度不匹配：意图识别通常是句子级分类，往往收敛较快，实体识别是序列标注，通常收敛速度较慢且更难。如果直接相加，当意图识别已经收敛，而实体识别还在剧烈波动时，实体识别的梯度可能会破坏意图识别已经学到的特征表示，导致灾难性遗忘或震荡无法收敛。

  - 量纲与不确定性不统一：样本数量级的差异会导致(实体识别)NER的Loss天然比意图识别大很多倍。

  - 负迁移：如果两个任务在某些特征上存在冲突，强制共享底层参数并简单相加Loss，可能导致模型在两个任务上都表现不如单独训练。

- 如何处理训练不平衡？

  - 引入可学习的权重系数，不要直接相加，而是给每个Loss加上权重 Ltotal = λ1 * Lintent + λ2 * Lentity

    - 人工调参如果Lentity ≈ 10 * Lintent，则设置λ1=10, λ2=1强行拉平量级。

    - 动态加权，将权重λ视为可学习的参数，让模型自己根据任务的学习难度动态调整权重。

- 梯度裁剪与归一化

  - GradNorm(torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)):
  
    - GradNorm是一种梯度归一化技术，核心目标是解决多任务学习中不同任务梯度幅度差异过大的问题，让模型在多个任务上均衡学习，避免某一任务的梯度主导训练过程。

- 分阶段训练
  
  - 预训练阶段

  - 微调阶段

  - 交替训练

- Loss截断或标准化
 
  - 在每个Epoch开始时，计算两个任务在验证集上的平均Loss，动态调整当前的λ，使得 λ1 * Lintent平均 ≈ λ2 * Lentity平均
