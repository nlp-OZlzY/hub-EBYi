## DeepSeek-V3 相比 V2 的改进

**架构层面**

- 沿用了 V2 的 MLA（Multi-head Latent Attention），继续通过低秩压缩 KV 缓存来降低推理显存开销，每 token 的 KV Cache 压缩到约 70KB，而传统 MHA 需要约 512KB
- MoE 部分从 V2 的辅助损失路由改为无辅助损失的负载均衡策略，用 bias term 代替额外的损失函数来平衡专家负载，避免了辅助损失对主任务性能的干扰
- MoE 进一步细化为更小粒度的专家切分，并增加了共享专家（shared expert），确保通用知识始终可用
- 模型规模从 V2 的 236B 总参 / 21B 激活扩大到 671B 总参 / 37B 激活，包含 256 个路由专家和 1 个共享专家

**训练创新**

- 引入 Multi-Token Prediction（MTP），模型同时预测未来多个 token，训练信号更丰富，数据效率更高，推理阶段还能用于推测解码实现约 1.8 倍加速
- 业界首次在大规模训练中成功采用 FP8 混合精度，通过 per-tile quantization 和高精度累加控制精度损失，大幅压缩训练成本
- 预训练数据从 V2 的 8.1T tokens 扩充到 14.8T tokens，但整个训练仅消耗约 2.788M H800 GPU 小时，折算下来总成本约 557 万美元，相比同级别模型极其经济（Llama 3.1 405B 用了 30.84M GPU 小时）
- 整个 14.8T token 的训练过程实现了零中断、零回滚

**推理能力增强**

- 通过从 DeepSeek-R1 进行知识蒸馏，把 reasoning 模型的长思维链能力压缩进通用模型，在数学和编程等推理密集型任务上提升显著
- 性能上与 GPT-4o、Claude 3.5 Sonnet 等闭源模型持平甚至在部分基准上超越，同时保持开源
- 支持 128K 上下文长度，配合 MLA 的极小 KV Cache，在长上下文推理场景下效率优势明显
