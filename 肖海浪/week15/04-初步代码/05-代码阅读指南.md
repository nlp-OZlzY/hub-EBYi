```python
@router.post("/upload/document", response_model=UploadResponse)
async def upload_document(file: UploadFile):
    # 1. 校验文件类型
    ext = os.path.splitext(file.filename)[1].lower()
    if ext not in settings.ALLOWED_EXTENSIONS:
        raise HTTPException(status_code=400, detail="仅支持 pdf/docx/txt 格式")

    # 2. 读取文件内容
    content = await file.read()

    # 3. 校验文件大小
    if len(content) == 0:
        raise HTTPException(status_code=400, detail="文件不能为空")

    # 4. 调用Service处理

    return UploadResponse(data=result)
```

**阅读要点**：
- `@router.post()` 定义POST接口
- `UploadFile` 是FastAPI的文件上传类型
- `HTTPException` 抛出HTTP错误
- 接口层只做参数校验，业务逻辑交给Service层

#### 8. services/file_service.py（文件管理服务）

**作用**：处理文件保存、数据库操作、Kafka消息发送。

**关键代码**：
```python
class FileService:
    def save_and_publish(self, filename: str, content: bytes) -> dict:
        # 1. 生成唯一文件名（使用UUID避免冲突）
        ext = os.path.splitext(filename)[1]
        save_name = f"{uuid.uuid4()}{ext}"
        save_path = os.path.join(settings.UPLOAD_DIR, save_name)

        # 2. 保存文件到本地
        with open(save_path, "wb") as f:
            f.write(content)

        # 3. 创建数据库记录
        with Session() as session:
            record = File(filename=filename, filepath=save_path, filestate="已上传")
            session.add(record)
            session.commit()
            file_id = record.id

        # 4. 发送Kafka消息（触发离线解析）
        self.producer.send(settings.KAFKA_TOPIC, value={
            "file_name": filename,
            "file_path": save_path,
            "id": file_id
        })
        self.producer.flush()

        return {"file_id": file_id, "filename": filename, "filestate": "已上传"}
```

**阅读要点**：
- `uuid.uuid4()` 生成唯一标识符
- `Session()` 是数据库会话，使用 `with` 自动关闭
- `KafkaProducer.send()` 发送消息到Kafka

```python
class ChatService:
    def chat(self, question: str) -> Dict:
        # 1. 编码问题
        question_vector = self.encode_service.encode_text(question)

        # 2. Milvus检索
        search_results = self.search_service.search(question_vector, limit=5)

        # 3. 拼接上下文
        context = "\n".join([r["text"] for r in search_results])

        # 4. 调用LLM生成回答
        completion = self.llm_client.chat.completions.create(
            model=settings.QWEN_MODEL,
            messages=[
                {'role': 'system', 'content': 'You are a helpful assistant.'},
                {'role': 'user', 'content': RAG_PROMPT.format(question, context)}
            ],
        )
        answer = completion.choices[0].message.content

        return {"answer": answer, "sources": sources}
```

**阅读要点**：
- 这是核心业务流程：编码 → 检索 → 生成
- `RAG_PROMPT` 是提示词模板
- 调用Qwen-VL API生成回答

### 第四步：理解离线处理

#### 12. offline_precess_worker.py（离线Worker）

**作用**：消费Kafka消息，解析文档，编码存储。

**关键代码**：
```python
class OfflineWorker:
    def process_message(self, message: dict):
        file_name = message['file_name']
        file_path = message['file_path']
        file_id = message['id']

        # 1. 更新状态为解析中
        self.update_file_state(file_id, "解析中")

        # 2. 调用mineru解析文档
        markdown_path = self.parse_document(file_path)

        # 3. 编码并存储到Milvus
        self.encode_and_store(markdown_path, file_id, file_name, file_path)

        # 4. 更新状态为已解析
        self.update_file_state(file_id, "已解析")

    def run(self):
        """启动Worker，监听Kafka消息"""
        for message in self.consumer:
            self.process_message(message.value)
```

**阅读要点**：
- `KafkaConsumer` 监听消息队列
- `subprocess.check_output()` 调用外部命令（mineru）
- 状态机：已上传 → 解析中 → 已解析/解析失败

### 第五步：理解工具函数

#### 13. utils/text_splitter.py（文本切分）

**作用**：将长文本切分为小块。

**关键代码**：
```python
def split_text2chunks(lines: List[str], chunk_size: int = 256) -> List[str]:
    chunks = []
    for line in lines:
        line = line.strip()

        # 跳过空行
        if not line:
            continue

        # 跳过References标题
        if line == "# References":
            continue

        # 跳过引用行 [1] xxx
        if len(line) > 2 and line[0] == "[" and line[1].isdigit():
            continue

        # 合并或新建chunk
        if len(chunks) == 0:
            chunks.append(line)
        else:
            if len(chunks[-1]) <= chunk_size:
                chunks[-1] += "\n" + line
            else:
                chunks.append(line)

    return chunks
```

**阅读要点**：
- 按行处理，跳过无用内容
- 当chunk长度超过 `chunk_size` 时新建chunk

## 四、关键概念解释

### 4.1 什么是向量检索？

```
文本 "什么是RAG？"
    │
    ▼ bge编码
向量 [0.1, 0.2, ..., 0.5]  (512个数字)
    │
    ▼ 在Milvus中找最相似的向量
返回 Top-5 相似文档
```

### 4.2 什么是RAG？

```
RAG = Retrieval + Augmented + Generation
      检索        增强         生成

流程：
1. 用户提问
2. 检索相关文档（Retrieval）
3. 将文档作为上下文（Augmented）
4. 让LLM基于上下文生成回答（Generation）
```

### 4.3 为什么要用Kafka？

```
同步方式（不推荐）：
用户上传 → 等待解析(1分钟) → 返回结果
用户会等待很久

异步方式（推荐）：
用户上传 → 发Kafka消息 → 立即返回
Worker后台慢慢解析

好处：
- 用户不用等待
- 解析失败不影响上传
- 可以水平扩展Worker
```

## 六、常见问题

### Q1: 为什么有些文件有 `__init__.py`？

`__init__.py` 让Python把目录当作包（package），可以使用 `from xxx import yyy` 的方式导入。

### Q2: 为什么用 `with Session() as session`？

这是上下文管理器，确保数据库连接用完后自动关闭，避免连接泄漏。

### Q3: 为什么用Mock做测试？

Mock可以模拟外部依赖（如Milvus、Kafka、LLM），让测试不依赖真实服务，运行更快更稳定。

### Q4: 文件状态为什么要流转？

```
已上传 → 解析中 → 已解析
                  → 解析失败

用途：
- 前端可以显示处理进度
- 可以重试失败的任务
- 可以统计处理成功率
```
