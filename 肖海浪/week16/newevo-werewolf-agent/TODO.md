## Bug / 体验优化

### 6. API 错误前端友好提示
- **现状**：LLM 调用失败（429 限流、401 认证错误）时，前端显示 500 错误，无友好提示
- **目标**：后端捕获 LLM 错误返回结构化响应，前端显示可读的错误信息
- **优先级**：中

### 7. 自动播放速度可调
- **现状**：自动播放固定每 2 秒一步，LLM 响应慢时日志断断续续
- **目标**：根据上一步完成时间动态调整间隔，或提供速度滑块
- **优先级**：低

### 8. 游戏后期 Prompt 过长导致 LLM 响应变慢
- **已修复**（2026-05-28）：
  - [x] `_truncate_game_state` 截断对话历史（保留最近 12 条）
  - [x] 精简 prompt 模板，去掉冗余说明
- **优先级**：高（已修复）

### 9. LLM JSON 输出遵循能力不足（影响全局）
- **现状**：MiMo 模型对 JSON 格式遵循不够强，经常返回非 JSON 文本或 `"target": null`，导致：
  - 狼人不杀人（target: null → 跳过击杀）
  - 发言为空（content 缺失 → 前端显示空消息）
  - 投票跳过（target: null → 无人被投出）
  - 游戏无法正常推进
- **已修复**（2026-05-28）：
  - [x] prompt 加强 JSON 输出约束
  - [x] `_parse_json_output` 4 层容错解析
  - [x] target 为 null 时随机选合法目标
  - [x] content 为空时用 LLM 原始输出兜底
- **待验证**：实际运行效果，可能还需微调 prompt 或 fallback 策略
- **优先级**：高（已修复，待验证）
