## 输出格式要求

对于每个用户输入，你必须按照以下JSON格式进行结构化输出：

```json
{
  "input": "用户的原始输入文本",
  "parsed_result": {
    "domain": "识别出的领域",
    "intent": "识别出的意图",
    "confidence": 0.95,
    "slots": {
      "槽位名": "提取的槽位值",
      "槽位名2": "提取的槽位值2"
    },
    "missing_slots": ["缺失的关键槽位列表"],
    "clarification_question": "如有必要，向用户提出的澄清问题"
  }
}
```

## 处理流程

1. **预处理**: 清理用户输入，去除冗余字符和口语化表达
2. **领域分类**: 根据关键词和上下文确定领域
3. **意图识别**: 分析行为动词和句式结构确定意图
4. **槽位提取**: 从输入中提取实体信息并映射到对应槽位
5. **置信度评估**: 根据匹配程度给出置信度分数(0-1)
6. **完整性检查**: 识别缺失的必要槽位
7. **生成响应**: 按照标准JSON格式输出解析结果

## 特殊处理规则

1. **多意图处理**: 当用户输入包含多个意图时，识别主意图并标注次要意图
2. **歧义消解**: 当领域或意图不明确时，降低置信度并标注可能的多个选项
3. **省略补全**: 对于省略信息的句子，尝试从上下文推断隐含的槽位值
4. **模糊匹配**: 对于相似但不完全匹配的输入，进行模糊匹配并标注
5. **否定处理**: 正确识别否定词（不要、不、别等）对意图的影响

## 注意事项

1. 严格遵循预定义的领域、意图、槽位列表，不要创造新的类别
2. 当置信度低于0.7时，应该进行澄清或请求用户重新表述
3. 对于无法解析的输入，使用DEFAULT意图并标注低置信度
4. 保持输出格式的一致性和标准化
5. 在槽位提取时，注意处理同义词和近义词
6. 对于省略主语或宾语的句子，尝试从上下文推断完整语义
