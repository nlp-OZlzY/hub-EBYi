## 1. 什么是前后端分离？

前后端分离是一种软件架构模式，将应用程序的前端和后端逻辑分开开发和部署。在 06-stock-bi-agent 项目中，这种架构模式得到了清晰的体现：

### 前端部分
- **技术栈**：使用 Streamlit 构建的 Web 应用
- **实现文件**：`demo/streamlit_demo.py` 及相关目录下的文件
- **功能**：负责用户界面展示、用户交互处理、数据可视化等
- **运行方式**：作为独立的 Web 应用运行，与后端通过 HTTP 请求通信

### 后端部分
- **技术栈**：使用 FastAPI 构建的 RESTful API 服务
- **实现文件**：`main_server.py`、`routers/` 目录下的文件、`services/` 目录下的文件等
- **功能**：负责业务逻辑处理、数据存储、API 接口提供、大模型调用等
- **运行方式**：运行在 8000 端口，接收前端的 HTTP 请求并返回响应

### 前后端通信方式
- 前端通过 HTTP 请求调用后端的 API 接口
- 后端返回 JSON 格式的数据或使用 Server-Sent Events (SSE) 进行流式响应
- 例如，前端通过 `http://127.0.0.1:8000/v1/chat/` 接口与后端进行聊天交互

## 2. 历史对话如何存储，以及如何将历史对话作为大模型的下一次输入

### 历史对话存储

历史对话的存储采用了双层存储机制：

1. **关系型数据库存储**：
   - 使用 SQLite 数据库 (`assert/conversations.db` 和 `assert/sever.db`)
   - 主要表结构：
     - `ChatSessionTable`：存储对话会话信息，包括会话 ID、用户 ID、标题等
     - `ChatMessageTable`：存储具体的消息内容，包括消息 ID、对话 ID、角色、内容等
   - 实现函数：`append_message2db` 用于将消息添加到数据库

2. **会话状态管理**：
   - 使用 `AdvancedSQLiteSession` 来管理会话状态
   - 存储位置：`./assert/conversations.db`
   - 作用：维护对话的历史状态，为大模型提供上下文信息

### 历史对话作为大模型的下一次输入

历史对话作为大模型输入的流程如下：

1. **前端获取历史对话**：
   - 当用户打开聊天页面时，前端通过 `http://127.0.0.1:8000/v1/chat/get?session_id={session_id}` 接口获取历史对话
   - 前端将历史对话加载到 `st.session_state.messages` 中，用于界面展示

2. **后端处理历史对话**：
   - 后端通过 `get_chat_sessions` 函数从数据库中获取历史对话
   - 在调用大模型时，使用 `AdvancedSQLiteSession` 来管理会话状态，将历史对话作为上下文传递给大模型

3. **大模型使用历史对话**：
   - 在 `chat` 函数中，创建 `AdvancedSQLiteSession` 实例，传入 `session_id`
   - 调用 `Runner.run_streamed` 时，传入 `session` 参数，大模型会使用会话中的历史信息作为上下文

4. **新消息的处理**：
   - 当用户发送新消息时，前端将消息添加到 `st.session_state.messages` 中
   - 后端通过 `append_message2db` 函数将新消息存储到数据库中
   - 大模型基于历史对话和新消息生成回复

### 关键代码实现

1. **历史对话存储**：
   ```python
   def append_message2db(session_id: str, role: str, content: str) -> bool:
       with SessionLocal() as session:
           message_recod = session.query(ChatSessionTable.id).filter(ChatSessionTable.session_id == session_id).first()
           if message_recod:
               message_recod = ChatMessageTable(
                   chat_id=message_recod[0],
                   role=role,
                   content=content
               )
               session.add(message_recod)
               session.commit()
   ```

2. **历史对话获取**：
   ```python
   def get_chat_sessions(session_id: str) -> List[Dict[str, Any]]:
       with SessionLocal() as session:
           chat_messages: Optional[List[ChatMessageTable]] = session.query(ChatMessageTable) 
               .join(ChatSessionTable) 
               .filter(
               ChatSessionTable.session_id == session_id
           ).all()
   
           result = []
           if chat_messages:
               for record in chat_messages:
                   result.append({
                       "id": record.id, "create_time": record.create_time,
                       "feedback": record.feedback, "feedback_time": record.feedback_time,
                       "role": record.role, "content": record.content
                   })
   
           return result
   ```

3. **大模型使用历史对话**：
   ```python
   # openai-agent支持的session存储，存储对话的历史状态
   session = AdvancedSQLiteSession(
       session_id=session_id, # 与 系统中的对话id 关联，存储在关系型数据库中
       db_path="./assert/conversations.db",
       create_tables=True
   )
   
   result = Runner.run_streamed(agent, input=content, session=session) # 流式调用大模型
   ```

4. **前端加载历史对话**：
   ```python
   if "session_id" in st.session_state.keys() and st.session_state.session_id:
       data = requests.post("http://127.0.0.1:8000/v1/chat/get?session_id=" + st.session_state['session_id']).json()
   ```
